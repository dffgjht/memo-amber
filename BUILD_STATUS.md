## 🚀 推荐构建方案

### 方案一：使用 Android Studio (强烈推荐)

#### 优点
- ✅ 零配置，开箱即用
- ✅ 可视化界面，操作简单
- ✅ 自动处理所有依赖
- ✅ 内置模拟器测试

#### 步骤
1. 下载 Android Studio: https://developer.android.com/studio
2. 安装并启动
3. 选择 "Open" → 选择项目文件夹
4. 等待 Gradle 同步 (首次约5-10分钟)
5. 菜单: `Build` → `Build Bundle(s) / APK(s)` → `Build APK(s)`

#### 预计时间
- 下载安装: 15-30 分钟
- 首次构建: 5-10 分钟
- 后续构建: 1-2 分钟

---

### 方案二：使用 GitHub Actions (云端构建)

#### 优点
- ✅ 无需本地环境
- ✅ 自动化构建
- ✅ APK 自动下载

#### 步骤
1. 将项目推送到 GitHub
2. 在 Actions 页面手动触发工作流
3. 等待构建完成 (约5分钟)
4. 下载生成的 APK

#### 工作流配置
```yaml
# 已配置: .github/workflows/build-apk.yml
# 功能:
#   - 自动构建 Debug 和 Release APK
#   - 自动上传构建产物
#   - 支持 API 34
```

---

### 方案三：命令行构建

#### 环境要求
- Java JDK 17+
- Android SDK API 34
- Android Build Tools 34.0.0

#### 步骤
```bash
# 1. 安装 Java
sudo apt install openjdk-17-jdk  # Linux
brew install openjdk@17          # macOS

# 2. 安装 Android SDK
# 使用 Android Studio 或命令行工具

# 3. 设置环境变量
export ANDROID_HOME=/path/to/Android/Sdk
export PATH=$PATH:$ANDROID_HOME/platform-tools

# 4. 构建
./gradlew assembleDebug
```

#### 预计时间
- 环境配置: 30-60 分钟
- 首次构建: 5-10 分钟

## 🔧 故障排除

### 常见问题

**Q1: Gradle 同步失败**
```
A: 检查网络连接，或使用 VPN
   配置国内镜像源 (见 BUILD_ENVIRONMENT.md)
```

**Q2: 找不到 Java**
```
A: 安装 JDK 17 或更高版本
   设置 JAVA_HOME 环境变量
```

**Q3: 构建失败**
```
A: 清理并重新构建
   ./gradlew clean
   ./gradlew assembleDebug
```

**Q4: APK 安装失败**
```
A: 检查 Android 版本 (需要 7.0+)
   启用"未知来源"安装
   检查存储空间
```
